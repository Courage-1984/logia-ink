## Image Optimization Guidelines

### File Naming Convention
- Use lowercase with hyphens: `service-web-development.jpg`
- Be descriptive and specific
- Include location context when needed: `about-mission.jpg`

### Image Sizes Summary
- **Hero Backgrounds:** 1920x1080px (Landscape)
- **Section Backgrounds:** 1200x800px (Landscape)
- **Project Images:** 800x600px (Landscape)
- **Service Cards:** 800x600px (Landscape)
- **Value Icons:** 600x600px (Square)
- **Social Icons:** 64x64px (Square)
- **Decorative Graphics:** 400-500px (Square/Various)

### Format Recommendations
- **Photographs/Realistic:** JPG
- **Icons/Logos:** SVG (preferred) or PNG
- **Graphics with Transparency:** PNG or SVG
- **Complex Graphics:** JPG for photos, SVG for illustrations

### Color Palette for All Images
- Primary Background: Dark (#0a0a0a, #1a1a2e, #16213e)
- Accent Colors: Cyan (#00ffff), Magenta (#ff00ff)
- Secondary Accents: Green (#00ff00), Blue (#0066ff), Pink (#ff0080)
- Text: White (#ffffff), Light Gray (#b0b0b0)

## Notes for Implementation

1. **All images should be optimized** for web use (compress before upload)
2. **Use WebP format** when possible for better performance (convert from JPG/PNG)
3. **Provide fallback formats** (JPG/PNG) for older browsers
4. **Add alt text** to all images for accessibility
5. **Lazy load images** below the fold for better performance
6. **Use responsive images** with srcset for different screen sizes
7. **Test image loading** on slow connections
