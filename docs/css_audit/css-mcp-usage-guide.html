1. Introduction

What is this guide about

This comprehensive guide provides a systematic approach to auditing CSS and font usage in your codebase using Model Context Protocol (MCP) servers within Cursor IDE. You'll learn to identify unused styles, optimize performance, eliminate duplicates, and restructure your CSS architecture for better maintainability and performance.

Prerequisites

  • Cursor IDE with MCP server support enabled
  • Basic understanding of CSS and web performance concepts
  • Access to your project's codebase
  • Chrome browser for DevTools integration

Overview of MCP servers you'll use

This guide leverages seven powerful MCP servers, each serving specific roles in the audit process:

  • Filesystem MCP - Project structure analysis and file operations
  • Chrome DevTools MCP - Performance profiling and CSS coverage analysis
  • Context7 MCP - Latest documentation and best practices research
  • Brave Search MCP - Industry standards and solution research
  • Playwright MCP - Automated testing and cross-browser validation
  • Sequential Thinking MCP - Strategic planning and decision-making
  • Firecrawl MCP - External resource research and documentation scraping

Step 4.2: Specificity Analysis

Using Chrome DevTools MCP - Specificity Issues

# Analyze CSS specificity conflicts
Use tool: evaluate_script
Arguments: {
  "script": `
    // Function to calculate CSS specificity
    function calculateSpecificity(selector) {
      let specificity = [0, 0, 0, 0];
      // Count IDs
      specificity[1] = (selector.match(/#[\\w-]+/g) || []).length;
      // Count classes, attributes, pseudo-classes
      specificity[2] = (selector.match(/\\.[\\w-]+|\\[[^\\]]+\\]|:[\\w-]+/g) || []).length;
      // Count elements and pseudo-elements
      specificity[3] = (selector.match(/^[\\w-]+|\\s[\\w-]+/g) || []).length;
      return specificity.join('');
    }

    // Analyze all stylesheets
    const sheets = Array.from(document.styleSheets);
    const specificityReport = [];

    sheets.forEach(sheet => {
      try {
        Array.from(sheet.cssRules).forEach(rule => {
          if (rule.selectorText) {
            specificityReport.push({
              selector: rule.selectorText,
              specificity: calculateSpecificity(rule.selectorText),
              sheet: sheet.href || 'inline'
            });
          }
        });
      } catch (e) {
        console.warn('Cannot access stylesheet:', e);
      }
    });
    return specificityReport;
  `
}

# Detailed CSS loading analysis
Use tool: evaluate_script
Arguments: {
  "script": `
    // Analyze CSS loading performance
    const cssAnalysis = {
      totalStylesheets: document.styleSheets.length,
      externalSheets: 0,
      inlineStyles: 0,
      totalRules: 0,
      loadTimes: []
    };

    // Performance timing for CSS resources
    const resources = performance.getEntriesByType('resource');
    cssAnalysis.loadTimes = resources
      .filter(r => r.name.includes('.css'))
      .map(r => ({
        name: r.name,
        duration: r.duration,
        transferSize: r.transferSize,
        encodedBodySize: r.encodedBodySize

    // Count total CSS rules
    Array.from(document.styleSheets).forEach(sheet => {
      try {
        cssAnalysis.totalRules += sheet.cssRules.length;
        if (sheet.href) {
          cssAnalysis.externalSheets++;
        } else {
          cssAnalysis.inlineStyles++;
        }
      } catch (e) {
        console.warn('Cannot access stylesheet rules:', e);
      }
    });

    return cssAnalysis;
  `
}

CSS Coverage and Usage Analysis

Playwright MCP - Comprehensive CSS Coverage

# Advanced CSS coverage analysis with user interaction simulation
Use tool: run_test
Arguments: {
  "script": `
    const browser = await playwright.chromium.launch();
    const page = await browser.newPage();

    // Start CSS coverage
    await page.coverage.startCSSCoverage();

    // Navigate to application
    await page.goto('http://localhost:3000');
    await page.waitForLoadState('networkidle');

    // Simulate user interactions to trigger dynamic styles
    const interactionPlan = [
      () => page.hover('[data-testid="menu-button"]'),
      () => page.click('[data-testid="menu-button"]'),
      () => page.fill('input[type="search"]', 'test query'),
      () => page.click('button[type="submit"]'),
      () => page.evaluate(() => window.scrollTo(0, document.body.scrollHeight))
    ];

    for (const interaction of interactionPlan) {
      await interaction();
      await page.waitForTimeout(500);
    }

    // Stop coverage and analyze
    const coverage = await page.coverage.stopCSSCoverage();
    const coverageReport = coverage.map(entry => {
      const usedBytes = entry.ranges.reduce((used, range) =>
        used + (range.end - range.start), 0
      );
      const totalBytes = entry.text.length;
      const usagePercentage = (usedBytes / totalBytes) * 100;

      return {
        url: entry.url,
        totalBytes,
        usedBytes,
        unusedBytes: totalBytes - usedBytes,
        usagePercentage: Math.round(usagePercentage * 100) / 100,
        unusedRanges: entry.ranges
      };
    });

    await browser.close();
    return coverageReport;
  `,
  "timeout": 60000
}

5. CSS Audit Checklist

File Organization & Structure

  • CSS files are logically organized (components, utilities, base, etc.)
  • Consistent naming conventions are used throughout
  • Import statements are organized and optimized
  • No circular dependencies between CSS modules
  • Separate concerns (layout, typography, colors, animations)

Code Quality

  • No duplicate CSS declarations
  • Consistent indentation and formatting
  • Meaningful class and ID names
  • Appropriate CSS selector specificity
  • No overly qualified selectors
  • CSS custom properties used for repeated values
  • Vendor prefixes are necessary and up-to-date

Performance

  • No unused CSS rules (verified through coverage analysis)
  • CSS files are minified in production
  • Critical CSS is inlined for above-the-fold content
  • Non-critical CSS is loaded asynchronously
  • CSS bundle size is optimized
  • HTTP/2 optimization considered for multiple files

Maintainability

  • Modular CSS architecture implemented
  • Component-scoped styles prevent global conflicts
  • Design system tokens are used consistently
  • CSS documentation is present and up-to-date
  • Build process includes CSS linting

Browser Compatibility

  • CSS features are supported in target browsers
  • Fallbacks provided for unsupported features
  • Progressive enhancement implemented
  • Cross-browser testing completed

Accessibility

  • Sufficient color contrast ratios maintained
  • Focus indicators are visible and accessible
  • Text remains readable when zoomed to 200%
  • No CSS-only interactive elements
  • Reduced motion preferences respected

6. Font Audit Checklist

Font File Optimization

  • Using modern font formats (WOFF2, then WOFF fallback)
  • Font files are compressed and optimized
  • Unnecessary font weights and styles removed
  • Font subsetting implemented for reduced file sizes
  • Variable fonts considered where appropriate

Font Loading Strategy

  • font-display property set appropriately (swap, optional, etc.)
  • Critical fonts preloaded with <link rel="preload">
  • Font loading strategy prevents FOIT (Flash of Invisible Text)
  • System font fallbacks defined
  • Font loading monitored and optimized

Performance Metrics

  • First Contentful Paint (FCP) not delayed by fonts
  • Largest Contentful Paint (LCP) optimized
  • Cumulative Layout Shift (CLS) minimal from font swapping
  • Total font file size under performance budget

Font Declarations

  • @font-face declarations properly structured
  • Font family names consistently used
  • Unicode ranges specified where needed
  • Font format hints provided
  • CORS headers configured for cross-origin fonts

Typography System

  • Consistent typography scale defined
  • Line heights optimized for readability
  • Font pairing follows design system
  • Responsive typography implemented
  • Text rendering optimized (text-rendering, font-smooth)

8. Performance Optimization Strategies

Critical CSS Extraction

üí° Strategy: Identify and inline CSS required for above-the-fold content rendering.

MCP Tools: Chrome DevTools for viewport analysis, Playwright for automated critical path detection

Chrome DevTools MCP - Critical CSS Detection

# Analyze above-the-fold content
Use tool: evaluate_script
Arguments: {
  "script": `
    // Get viewport dimensions
    const viewport = {
      width: window.innerWidth,
      height: window.innerHeight
    };

    // Find elements in critical path
    const criticalElements = Array.from(document.querySelectorAll('*'))
      .filter(el => {
        const rect = el.getBoundingClientRect();
        return rect.top < viewport.height && rect.left < viewport.width;
      })
      .map(el => ({
        tagName: el.tagName,
        className: el.className,
        id: el.id,
        position: el.getBoundingClientRect()
      }));

    return { viewport, criticalElements };
  `
}

Font Preloading Optimization

Chrome DevTools MCP - Font Preload Analysis

# Check current font preloading strategy
Use tool: evaluate_script
Arguments: {
  "script": `
    // Analyze font preloading
    const fontPreloads = Array.from(document.querySelectorAll('link[rel="preload"][as="font"]'));
    const fontFaces = [];

    // Check @font-face declarations
    Array.from(document.styleSheets).forEach(sheet => {
      try {
        Array.from(sheet.cssRules).forEach(rule => {
          if (rule.constructor.name === 'CSSFontFaceRule') {
            fontFaces.push({
              family: rule.style.fontFamily,
              weight: rule.style.fontWeight,
              style: rule.style.fontStyle,
              display: rule.style.fontDisplay,
              src: rule.style.src
            });
          }
        });
      } catch (e) {
        console.warn('Cannot access stylesheet:', e);
      }
    });

    return {
      preloadedFonts: fontPreloads.map(link => ({
        href: link.href,
        crossorigin: link.crossOrigin
      })),
      declaredFonts: fontFaces,
      recommendations: {
        missingPreloads: fontFaces.filter(font =>
          !fontPreloads.some(preload => preload.href.includes(font.family))
        )
      }
    };
  `
}

    const browser = await playwright.chromium.launch();
    const page = await browser.newPage();

    // Start CSS coverage
    await page.coverage.startCSSCoverage();
    await page.goto('http://localhost:3000');

    // Test various application states
    const testStates = [
      { action: 'hover', selector: '.interactive-element' },
      { action: 'click', selector: '.menu-toggle' },
      { action: 'focus', selector: 'input' },
      { action: 'scroll', position: { x: 0, y: 1000 } },
      { action: 'resize', viewport: { width: 768, height: 1024 } }
    ];

    for (const test of testStates) {
      switch (test.action) {
        case 'hover':
          await page.hover(test.selector);
          break;
        case 'click':
          await page.click(test.selector);
          break;
        case 'focus':
          await page.focus(test.selector);
          break;
        case 'scroll':
          await page.evaluate((pos) => window.scrollTo(pos.x, pos.y), test.position);
          break;
        case 'resize':
          await page.setViewportSize(test.viewport);
          break;
      }
      await page.waitForTimeout(1000);
    }

    const coverage = await page.coverage.stopCSSCoverage();
    await browser.close();

    return coverage.map(entry => ({
      url: entry.url,
      usedPercentage: (entry.ranges.reduce((sum, range) =>
        sum + (range.end - range.start), 0) / entry.text.length) * 100
    }));

# Multiple run performance analysis
Use tool: run_test
Arguments: {
  "script": `
    const results = [];
    const runs = 5;

    for (let i = 0; i < runs; i++) {
      const browser = await playwright.chromium.launch();
      const page = await browser.newPage();

      // Clear cache between runs
      await page.context().clearCookies();

      const startTime = Date.now();
      await page.goto('http://localhost:3000', { waitUntil: 'networkidle' });

      const metrics = await page.evaluate(() => ({
        fcp: performance.getEntriesByType('paint')
          .find(entry => entry.name === 'first-contentful-paint')?.startTime,
        lcp: performance.getEntriesByType('largest-contentful-paint')[0]?.startTime,
        cls: performance.getEntriesByType('layout-shift')
          .reduce((sum, entry) => sum + entry.value, 0)
      }));

      results.push({
        run: i + 1,
        loadTime: Date.now() - startTime,
        ...metrics
      });

      await browser.close();
    }

    // Calculate averages
    const averages = {
      loadTime: results.reduce((sum, r) => sum + r.loadTime, 0) / runs,
      fcp: results.reduce((sum, r) => sum + (r.fcp || 0), 0) / runs,
      lcp: results.reduce((sum, r) => sum + (r.lcp || 0), 0) / runs,
      cls: results.reduce((sum, r) => sum + (r.cls || 0), 0) / runs
    };

    return { individual: results, averages };
  `,
  "timeout": 120000
}

Performance Budget Validation

# Automated performance budget enforcement
Use tool: run_test
Arguments: {
  "script": `
    const budget = {
      totalCSS: 100000, // 100KB
      criticalCSS: 14000, // 14KB
      totalFonts: 200000 // 200KB
    };

    const browser = await playwright.chromium.launch();
    const page = await browser.newPage();
    await page.goto('http://localhost:3000');

    const resources = await page.evaluate(() => {
      return performance.getEntriesByType('resource')
        .filter(r => r.name.includes('.css') || r.name.includes('font'))
        .map(r => ({
          name: r.name,
          size: r.transferSize,
          type: r.name.includes('.css') ? 'css' : 'font'
        }));
    });

    const totals = {
      css: resources.filter(r => r.type === 'css')
        .reduce((sum, r) => sum + r.size, 0),
      fonts: resources.filter(r => r.type === 'font')
        .reduce((sum, r) => sum + r.size, 0)
    };

    const violations = [];
    if (totals.css > budget.totalCSS) {
      violations.push('CSS budget exceeded');
    }
    if (totals.fonts > budget.totalFonts) {
      violations.push('Font budget exceeded');
    }

    await browser.close();
    return { totals, budget, violations };
  `
}
